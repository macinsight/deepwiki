# Contributing to DeepWiki

Welcome! We're building this wiki collaboratively. Here's how you can help.

## Getting Started

1. **Fork the repository** on GitHub
2. **Clone your fork** locally: `git clone https://github.com/YOUR-USERNAME/deepwiki.git`
3. **Create a new branch** for your contribution: `git checkout -b your-contribution-name`
4. **Make your changes** and commit with clear messages
5. **Push to your fork** and open a Pull Request

## Content Guidelines

### Article Structure
- Start with a clear introduction
- Use headings to organize content (H2 for major sections)
- Keep language accessible to new and veteran players alike
- Include examples where helpful

### What to Write About
- **Lore**: In-universe history, factions, and world-building
- **Characters**: Notable NPCs, companions, and character guides
- **Locations**: Areas, dungeons, and world geography
- **Mechanics**: Game systems, combat, crafting, etc.

### Quality Standards

**Before submitting:**
- [ ] Proofread for typos and grammar
- [ ] Verify information is accurate and current
- [ ] Link to related articles where appropriate
- [ ] Add appropriate topic labels
- [ ] Mark with `[WIP]` in the title if incomplete
- [ ] Add `[SPOILERS]` warning if content contains major plot points

### Labeling Your Content
- Use `stub` for incomplete articles
- Use `wip` for work in progress
- Use `needs-review` when ready for editorial review
- Use `spoilers` if your content reveals major story points
- Use `outdated` if updating old information

## Review Process

1. Submitters add the `needs-review` label
2. Community reviewers check for:
   - Accuracy and factual correctness
   - Tone consistency with existing articles
   - Proper formatting and links
   - Spoiler warnings where needed
3. Once approved, the article receives the `verified` label
4. Content is merged and automatically deployed

## Code of Conduct

- Be respectful and collaborative
- Focus on improving the wiki, not personal disputes
- Assume good faith in others' contributions
- Give constructive feedback

## Questions?

- Check existing articles and pull requests first
- Open a discussion if you have broader questions
- Ask in PR comments if you need clarification on feedback

Thank you for helping build this wiki! 📖
